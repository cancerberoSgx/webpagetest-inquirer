Check this if you want to see the results based on the webpagetest second/repeat view.
Each webpagetest sample consists in a first page load result where the browser cache is empty and in a second run where the browser's cache already may have some resources cached locally. So this second run or repeat run is often faster than the first run.
If the page process a lot of css/js the differences will be more noticeable here (and more in slower devices) because is less networking.
